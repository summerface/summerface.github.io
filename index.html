html严格模式与混杂模式

严格模式与混杂模式

  • Doctype可声明三种DTD类型，分别表示严格版本、过渡版本以及基于框架的 HTML 文档。
  • 当浏览器厂商开始创建与标准兼容的浏览器时，他们希望确保向后兼容性。为了实现这一点，他们创建了两种呈现模式：标准模式和混杂模式。
  • 标准模式中，浏览器以其支持的最高标准呈现网页。
  • 在混杂模式中，页面以一种比较宽松的向后兼容的方式显示。混杂模式通常模拟老式浏览器的行为以防止老站点无法工作。

差异

对于这两种模式之间的差异，最显著的一个例子与Windows上IE专有的盒模型有关。在IE 6出现时，在标准模式中使用的是正确的盒模型，在混杂模式中使用的则是老式的专有盒模型。为了维持对IE 5和更低版本的向后兼容性，Opera 7和更高版本也在混杂模式中使用有缺点的IE盒模型。
呈现方面的其他差异比较小，而且是与特定浏览器相关的，包括对于十六进制颜色值不需要#号、假设CSS中没有指定单位的长度的单位是像素，以及在使用关键字时将字号增加一级。

触发

浏览器根据DOCTYPE是否存在以及使用的哪种DTD来选择要使用的呈现方法。如果XHTML文档包含形式完整的DOCTYPE，那么它一般以标准模式 呈现。对于HTML 4.01文档，包含严格DTD的DOCTYPE常常导致页面以标准模式呈现。包含过渡DTD和URI的DOCTYPE也导致页面以标准模式呈现，但是有过 渡DTD而没有URI会导致页面以混杂模式呈现。DOCTYPE不存在或形式不正确会导致HTML和XHTML文档以混杂模式呈现。
根据DOCTYPE是否存在选择呈现模式，被称为DOCTYPE切换或DOCTYPE侦测。并非所有浏览器都采用这些规则，但是这些规则很好地说明了DOCTYPE切换的工作方式。
DOCTYPE切换是浏览器用来区分遗留文档和符合标准的文档的手段。无论是否编写了有效的CSS，如果选择了错误的DOCTYPE，那么页面就将以混杂 模式呈现，其行为就可能会有错误或不可预测。因此，一定要在站点的每个页面上包含形式完整的DOCTYPE声明，并且在使用HTML时选择严格的DTD。
浏览器根据DOCTYPE是否存在以及使用的哪种DTD来选择要使用的呈现方法。

  • 如果XHTML、HTML 4.01文档包含形式完整的DOCTYPE，那么它一般以标准模式呈现。
  • 包含过渡DTD和URI的DOCTYPE也导致页面以标准模式呈现，但是有过渡DTD而没有URI会导致页面以混杂模式呈现。
  • DOCTYPE不存在或形式不正确会导致HTML和XHTML文档以混杂模式呈现。
    html5既然没有DTD，也就没有严格模式与宽松模式的区别，html5有相对宽松的语法，实现时，已经尽可能大的实现了向后兼容。

css浮动与定位详细

浮动

会以某种方式将浮动元素从文档的正常流中删除不过它还是会影响布局，采用css的特有方式，浮动元素几乎自成一派，不过他们对文档的其余部分还是有一定影响。一个元素浮动时，其他的元素会环绕该元素。浮动元素的外边距不会与周围的外边距合并。

浮动元素的规则：

  • 浮动元素的左（右）外边界不能超出其包含快的左（右）内边界。
  • 浮动元素的左（或右）外边界必须是源文档中之前出现的左浮动（或右浮动）元素的右（或左）外边界，除非出现浮动元素的顶端在出现浮动元素的底端下面。
  • 左浮动元素的右外边界不会再起右边右浮动元素的左外边界的右边。一个右浮动元素的左外边界不会在其左边左浮动元素右外边界的左边。若出现此种情况，将要求第二个图像乡下浮动，知道其顶端在左浮动图像的底端之下。
  • 一个浮动元素的顶端不能比其父元素的内顶端更高，如果一个浮动元素在两个合并外边距之间，放置这个浮动元素时就好像在两个元素之间有一个块级父元素。
  • 浮动元素的顶端不能比之前所有浮动元素或块级元素的顶端更高。
  • 如果原文档中一个浮动元素之前出现另一个元素，浮动元素的顶端不能比包含该元素所生成狂的任何行框的顶端更高。
  • 动元素不能超出其包含元素的边界：左（右）浮动元素的左边（右边）有另一个浮动元素，前者的右外边界不能在其包含块的右（左）边界的右边（左边）。
  • 浮动元素必须尽可能高的放置。
  • 左浮动元素必须向左晶科能源，右浮动元素则必须向右尽可能远，位置越高，就会越向右或向左浮动的远。

实用行为

  • 浮动元素会延伸，从而包含其所有后代浮动元素，所以，通过将父元素设置为浮动元素，就可以把浮动元素包含在其父元素内。
  • 负外边距可能导致浮动元素移到父元素的外面。

行内框与一个浮动元素重叠时，其边框背景和内容都在该浮动元素之上显示。
块框与一个浮动元素重叠时，其边框背景在该浮动元素之下，但是内容在浮动元素之上。

清除浮动行为：

利用clear属性完成

  • clear:left 推过所有的左浮动元素，不过依然允许浮动元素出现在元素右边。
  • clear:both 不会与任何浮动元素在同一行。
  • clear:none 允许元素浮动到另一个元素的任意两边。

clear工作原理

  • css1.0 2.0:增加元素上外边距。
  • css2.1 在元素上外边距上方增加清除区域，元素的上外边距不变。

定位

position:static relative absolute fixed inherit，初始值为static。

  • relative:元素框偏移某个距离，元素仍保持其未定位前的形状，它原本所占的空间仍保留。
  • positive:元素原先在正常文档流中的空间关闭，元素定位后生成一个块级框，不论原来在正常文档流中生成什么框。
  • fixed:类似于absolute，不过其包含块是文档本身。

偏移属性

relative absolute fixed使用了4个属性描述偏移： top,right,bottom,left
偏移属性定义了距离包含块相应边的偏移而不是距包含块左上角的偏移。
偏移属性可设置为长度值，百分数值和auto

如果使用top,right,bottom,left来描述4个边的放置位置，那么元素的宽度和高度将由这些偏移隐含确定。但是如果向元素增加外边距边框或者内边距，此时若为height和width指定了值，则情况不同，因为向内容区增加了内边距，而且内边距的大小由height和width的值决定。

使用最小最大属性（min-height,max-height）的好处是，可以相对安全的混合使用不同的单位。
如果绝对定位元素溢出内容框，且overflow为hidden，通过clip可改变剪裁区域形状,rect(top,right,bottom,left)
rect的值不是边偏移，而是距离元素左上角的位置，不允许有百分数，可以设置负长度值

元素可见性

visibility:visible,hidden,collapses,inherit
处于hidden时，虽然不可见，但是不影响文档布局，与display:none有区别。使元素不可见，但是不删除其内容框

绝对定位

绝对定位的包含块是最近的position值部位static的祖先元素。通常会选择一个元素作为绝对定位元素的包含块，将position设置为relative且没有偏移
元素绝对定位时，也会为后代元素建立一个包含块

如果文档可滚动，定位元素会随着它滚动，只要绝对定位元素不是固定定位元素的后代。
如果一个绝对定位元素的top或者left或者right属性设置为auto，则其位置为其静态位置，即元素在正常流中原本的位置

webpack简介

webpack是一个打包工具，而不是像RequireJS或SeaJS这样的模块加载器，通过使用Webpack，能够像Node.js一样处理依赖关系，然后解析出模块之间的依赖，将代码打包。

安装

通过npm install -g webpack安装webpack，当然也可以通过gulp来处理webpack任务，如果使用gulp的话就npm install –save-dev gulp-webpack。

Webpack还有很多其他的配置，具体可以参照它的配置文档

css hack

CSS hack由于不同厂商的浏览器，比如Internet Explorer,Safari,Mozilla Firefox,Chrome等，或者是同一厂商的浏览器的不同版本，如IE6和IE7，对CSS的解析认识不完全一样，因此会导致生成的页面效果不一样，得不到我们所需要的页面效果。 这个时候我们就需要针对不同的浏览器去写不同的CSS，让它能够同时兼容不同的浏览器，能在不同的浏览器中也能得到我们想要的页面效果。

css hack分类

CSS属性前缀法、选择器前缀法以及IE条件注释法，实际项目中CSS Hack大部分是针对IE浏览器不同版本之间的表现差异而引入的。 -

  • 属性前缀法(即类内部Hack):例如 IE6能识别下划线”“和星号” “，IE7能识别星号” “，但不能识别下划线”“，IE6~IE10都认识”\9”，但firefox前述三个都不能认识。
  • 选择器前缀法(即选择器Hack)：例如 IE6能识别html .class{}，IE7能识别+html .class{}或者*:first-child+html .class{}。
  • IE条件注释法(即HTML条件注释Hack)：针对所有IE(注：IE10+已经不再支持条件注释)： ，针对IE6及以下版本： 。这类Hack不仅对CSS生效，对写在判断语句里面的所有代码都会生效。
CSS hack方式一：条件注释法

这种方式是IE浏览器专有的Hack方式，微软官方推荐使用的hack方式。

CSS hack方式二：类内属性前缀法

属性前缀法是在CSS样式属性名前加上一些只有特定浏览器才能识别的hack前缀，以达到预期的页面展现效果。
说明：在标准模式中

  • “-″减号是IE6专有的hack
  • “\9″ IE6/IE7/IE8/IE9/IE10都生效
  • “\0″ IE8/IE9/IE10都生效，是IE8/9/10的hack
  • “\9\0″ 只对IE9/IE10生效，是IE9/10的hack
CSS hack方式三：选择器前缀法

选择器前缀法是针对一些页面表现不一致或者需要特殊对待的浏览器，在CSS选择器前加上一些只有某些特定浏览器才能识别的前缀进行hack。
目前最常见的是

html 前缀只对IE6生效
+html +前缀只对IE7生效
@media screen\9{…}只对IE6/7生效
@media \0screen {body { background: red; }}只对IE8有效
@media \0screen\,screen\9{body { background: blue; }}只对IE6/7/8有效
@media screen\0 {body { background: green; }} 只对IE8/9/10有效
@media screen and (min-width:0\0) {body { background: gray; }} 只对IE9/10有效
@media screen and (-ms-high-contrast: active), (-ms-high-contrast: none) {body { background: orange; }} 只对IE10有效
等等

jquery-mobile content&widgets

jquery mobile包含的touch组件包括：

           form elements, collapsibles, collapsible sets (accordions), popups, dialogs, responsive tables等。

1. listviews

    data-role=”listview” data-inset=”true” data-filter=”true” data-type=”search”

2. formElements

    表单元素不需要data-role属性，但所有的表单一定要包含在form标签内部

3. theme

    data-theme=”b” 把本页面内所有的风格变为黑色。

4. buttons

常用类：

ui-btn ui-corner-all ui-icon-delete ui-btn-icon-notext ui-shadow ui-btn-inline ui-btn-a ui-btn-b
ui-mini ui-btn-icon-left ui-btn-icon-right ui-btn-icon-top ui-btn-icon-bottom ui-btn-icon-notext
ui-state-disabled

按钮种类
  • input标签，type=”reset” type=”submit” type=”button”
  • data-enhanced,如果设置为 “true”, (default) jQuery Mobile 会自动为页面添加样式，使其更适合移动设备。如果设置为 “false”，则框架不会设置页面的样式。

5. 选择框

  • data-mini=”true” 设置大小
  • fieldset加上data-role=”controlgroup”属性，设置为选择列表
  • fieldset加上data-type=”horizontal”属性，设置为水平排列列表
  • fieldset加上data-iconpos=”right”属性，设置label的位置
  • fieldset加上data-theme=”b”,设置整体风格
  • input加上disabled=””,设置为不可用选择框

6. 单选框

  • fieldset加上data-role=”controlgroup”属性，设置为选择列表
  • fieldset加上data-type=”horizontal”属性，设置为水平排列列表
  • fieldset加上data-iconpos=”right”属性，设置label的位置
  • fieldset加上data-theme=”b”,设置整体风格
  • input加上disabled=””,设置为不可用选择框

7. collapsible

  • data-theme=”b” data-content-theme=”b” 给标题和内容分别设置theme
  • 默认展开 data-collapsed=”false”
  • 设置延展图标data-collapsed-icon=”carat-d” data-expanded-icon=”carat-u”
  • 不显示边界框 data-inset=”false”

8. filterable

对任何元素都可以进行搜索
data-filter-reveal=”true”属性 当搜索框为空时 数据显示区域为空，输入相应条件时 显示搜索结果。
data-enhanced=”true”属性，可以默认filter框的value值，预xuanran数据显示区域数据。
自定义filter回调函数

9. 开关滑动条

常用属性：

data-on-text=”Light” data-off-text=”Dark” data-wrapper-class=”custom-label-flipswitch” data-wrapper-class=”custom-size-flipswitch”

jquery-mobile学习

jquery-mobile学习笔记

  1. 加载页面
           通过调用jquery mobile的loadPage()方法，可以将人意页面加载到当前的DOM中，执行完该方法之后，pagecreate事件重新被处罚，因为整个DOM的结构发生了变化，制定的目标页面已注入当前文档。
  2. 对话框
           在指向页面链接元素中添加”data-rel”属性为”dialog”，则单击该链接的时候，打开的页面将以一个对话框的形式展示在浏览器中。单击对话框任意链接，对话框自动关闭，并以回退形式至上一页面。也可以在对话框中创建取消按钮，通过设置元素属性或编写js代码关闭对话框。
           在对话框添加链接按钮”data-rel”=”back”可关闭对话框。
           或者js代码 $(“.ui-dialog”).dialog(“close”);
  3. 工具栏与格式化内容
           ”data-add-back-btn”=”back”设置为回退按钮
           ”data-back-btn－text”=”*“设置回退按钮的文字为＊＊＊
  4. “page”容器元素的“data-add-back-btn”为true时表示切换到该容器，头部栏显示默认的回退按钮。或者$.mobile.page.prototype.options.backBtnText=”*“
    data-icon=”arrow-l” “arrow-r” 分别显示左右按钮图标.
    设定按钮位置 添加类别属性 class=”ui-btn-left” “ui-btn-right”.
  5. 导航栏
           将div标签的data-role设置为navbar便会产生一个导航栏容器，在该容器内，将ul元素设置导航栏的各子类导航按钮，每一行最多可以设置5个按钮，超出个数的按钮自动显示在下一行，另外，导航栏中的按钮可以引用系统图标，也可以自定义图标。
    class=”ui-btn-active”设置导航按钮为选中状态。
